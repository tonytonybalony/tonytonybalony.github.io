    Humans reason based on existing knowledge and draw conclusions. The concept of representing knowledge and drawing conclusions from it is also used in AI, and in this lecture we will explore how we can achieve this behavior.

    Knowledge-Based Agents

    These are agents that reason by operating on internal representations of knowledge.

    I will explain what does “reasoning based on knowledge to draw a conclusion” mean using the following example.

    1. If it didn’t rain, Harry visited Hagrid today.
    2. Harry visited Hagrid or Dumbledore today, but not both.
    3. Harry visited Dumbledore today.

    Based on these three sentences, we can draw two conclusions

    Harry did not visit Hagrid.
    It rained today.

    Sentence: A sentence is an assertion about the world in a knowledge representation language. A sentence is how AI stores knowledge and uses it to infer new information.

    Model

    Propositions are statements about the world that can be either true or false. However, knowledge about the world is represented in the truth values of these propositions. The model is all the truth-value assignments that provide information about the world.

    Model Example

    • Let P: “It is raining.”
    • Let Q: “It is Tuesday.”

    However, there are more possible models in this situation (another example, {P = True, Q = True}, where it is both raining and a Tuesday).

    In fact, the number of possible models is 2 to the power of the number of propositions. In this case, we had 2 propositions, so 2²=4 possible models.

    Knowledge Base

    KB

    The knowledge base is a set of sentences known by a knowledge-based agent. This is knowledge that the AI is provided about the world in the form of propositional logic sentences that can be used to make additional inferences about the world.

    Entailment $\models$

    Entailment is a logical relationship where, if a set of sentences (the knowledge base) is true, another sentence must also be true in every possible model. In other words, the knowledge base entails a conclusion if, whenever the knowledge base is true, the conclusion is also true.

    For example, $(P \models Q)$ means that in every model where P is true, Q is also true.

    Implication:

    Connecting truth values is about how the truth of one proposition affects the truth of another in a specific situation, often using truth tables. For example, $(P \rightarrow Q)$ is true except when P is true and Q is false. Truth value connection checks how statements relate in one situation (truth table).

    Entailments:

    A semantic relationship is about whether, in every possible model (interpretation of the world), the truth of one set of sentences guarantees the truth of another. It’s not just about matching truth values(like how implication does), but about the logical consequence across all possible scenarios. Semantic relationship checks if one statement must be true whenever another is true, in all possible worlds (models).

    Inference

    Inference is the process of deriving new conclusions from an existing knowledge base using logical rules.

    • To determine if $(KB \models α)$ (in other words, answering the question: “can we conclude that α is true based on our knowledge base”)
      • Enumerate all possible models.
      • If in every model where KB is true, α is true as well, then KB entails α $(KB \models α)$.

    Resolution

    Resolution is a powerful inference rule state that if one of two atomic propositions in an disjunction is false, the other has to be true. It relies on Complementary Literals, two of the same atomic propositions where one is negated and the other is not, such as P and ¬P.

    Clause

    A disjunction of literals.

    e.g. P ∨ Q ∨ R

    Clauses allow us to convert any logical statement into a Conjunction Normal Form(CNF, just like Product of Sum in boolean algebra), which is a conjunction of clauses, for example: (A ∨ B ∨ C) ∧ (D ∨ ¬E) ∧ (F ∨ G).

    Steps in Conversion of Propositions to Conjunctive Normal Form

    • Eliminate biconditionals
      • Turn (α ↔ β) into (α → β) ∧ (β → α).
    • Eliminate implications
      • Turn (α → β) into ¬α ∨ β.
    • Move negation inwards until only literals are being negated (and not clauses), using De Morgan’s Laws.
      • Turn ¬(α ∧ β) into ¬α ∨ ¬β

    Here’s an example of converting (P ∨ Q) → R to Conjunctive Normal Form:

    • (P ∨ Q) → R
    • ¬(P ∨ Q) ∨ R /Eliminate implication
    • (¬P ∧ ¬Q) ∨ R /De Morgan’s Law
    • (¬P ∨ R) ∧ (¬Q ∨ R) /Distributive Law

    At this point, we can run an inference algorithm on the conjunctive normal form. Occasionally, through the process of inference by resolution, we might end up in cases where a clause contains the same literal twice. In these cases, a process called factoring is used, where the duplicate literal is removed. For example, (P ∨ Q ∨ S) ∧ (¬P ∨ R ∨ S) allow us to infer by resolution that (Q ∨ S ∨ R ∨ S). The duplicate S can be removed to give us (Q ∨ R ∨ S).

    Resolving a literal and its negation, i.e. ¬P and P, gives the empty clause (). The empty clause is always false, and this makes sense because it is impossible that both P and ¬P are true. This fact is used by the resolution algorithm.

    • To determine if KB ⊨ α:
      • Check: is (KB ∧ ¬α) a contradiction?
        • If so, then KB ⊨ α.
        • Otherwise, no entailment.

    Proof by contradiction is a tool used often in computer science. If our knowledge base is true, and it contradicts ¬α, it means that ¬α is false, and, therefore, α must be true. More technically, the algorithm would perform the following actions:

    • To determine if KB ⊨ α:
      • Convert (KB ∧ ¬α) to Conjunctive Normal Form.
      • Keep checking to see if we can use resolution to produce a new clause.
      • If we ever produce the empty clause (equivalent to False), congratulations! We have arrived at a contradiction, thus proving that KB ⊨ α.
      • However, if contradiction is not achieved and no more clauses can be inferred, there is no entailment.

    Here is an example that illustrates how this algorithm might work:

    • Does (A ∨ B) ∧ (¬B ∨ C) ∧ (¬C) entail A?
    • First, to prove by contradiction, we assume that A is false. Thus, we arrive at (A ∨ B) ∧ (¬B ∨ C) ∧ (¬C) ∧ (¬A).
    • Now, we can start generating new information. Since we know that C is false (¬C), the only way (¬B ∨ C) can be true is if B is false, too. Thus, we can add (¬B) to our KB.
    • Next, since we know (¬B), the only way (A ∨ B) can be true is if A is true. Thus, we can add (A) to our KB.
    • Now our KB has two complementary literals, (A) and (¬A). We resolve them, arriving at the empty set, (). The empty set is false by definition, so we have arrived at a contradiction.
