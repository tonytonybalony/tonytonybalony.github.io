    Search is finding a solution to a given problem. Search problems involve an initial state and a goal state, and returns a solution on how it gets from the former to the latter.

    Agent

    An entity that perceives its environment and acts upon that environment.

    State

    A configuration of an agent in its environment.
    Initial State is the state from which the algorithm starts.
    Goal State is the state at which the algorithm should stop.

    Actions

    The choices that can be made in a state.

    Transition Model

    A desciption of how a state moves from one to another, either as an algorithm or a set of rules.

    State Space

    The set of all State reachable from the initial state by any sequence. Graphically, it looks like all the states that connect each others by actions and forms a graph structure State Space.

    Uninformed First Search

    Search strategies that use no problem-specific knowledge.

    Breadth-First Search (BFS)

    Explore all nodes at one level before moving on to the next.
    Use queue to keep track of the nodes.
    Good for finding the shortest path in unweighted graphs.

    Depth-First Search (DFS)

    Explore all nodes at one path and head back to another path.
    Use stack to keep track of the nodes.
    Useful for tasks like topological sorting, cycle detection, or exploring all paths.

    They are 2 fundamental ways for traversing or searching data structures such as trees or graphs Lets look at this GIF.

    They practice of both of these algorithm shares the same concept, however BFS is applied into the frontier by Queue and DFS is applied into the frontier by Stack. Let me explain this by applying the left-hand side graph with DFS. First, look at the left-hand side of the GIF. Imagine there is a stack frontier and a node. The node first point at 1. Stack push 1 -> pop 1 and push 3 2. The node now points to 2. Stack pop 2 and push 5 and 4. The node points to 4. Stack pop 4 and the node now points 5. Stack pop 5 -> push 7 -> pop 7. The node now points to 3. Stack pop 3 -> push 6 -> pop 6 -> push 8 -> pop 8. Follow the sequence we pop, 1 -> 2 -> 4 -> 5 -> 7 -> 3 -> 6 -> 8 this is sequence is same as the one on the right functions. You may figure the right-hand side one out by applying it with a queue frontier by yourself.

    Informed Search

    Search strategy that uses problem-specific knowledge to find solutions more efficiently

    Greedy Best-First Search

    Search algorithm that expands the node that is closest to the goal, as estimated by a heuristic function

    Heuristic Function: It is a strategy used in algorithms to estimate the cost or distance to reach the goal, which can significantly improve problem-solving efficiency. It provides searching algorithms estimates of the most promising paths to a solution.

    With greedy best-first search, it will choose the path which cost the least to reach the goal, the right side with 11 blocks away.

    A* Search (A star Search)

    Search algorithm that expands node with the lowest value of g(n)+h(n), where g(n) = cost to reach node and h(n) = estimated cost to goal which is heuristic function.

    With A* Search, it get to choose the path with the least value of g(n)+h(n) which one is 15+6 and the other is 6+13. It then move on to the block with 13 and encounter another path swifting situation with 7+14 and 15+6, it chooses the current path and so on until hit the goal.

    Optimal Cases

    h(n) is admissible (never overestimates the true cost), and h(n) is consistent (for each node n and successor n’ with cost c, h(n) ≤ h(n’)+c ).
    Simply speaking, if I am at a current state, the next states heuristic value plus the cost for me to make that step should not increases. The better the heuristic function is, the better you will be able to solve the problem in the fewer states that you will have to explore.
